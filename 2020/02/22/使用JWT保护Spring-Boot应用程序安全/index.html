使用JWT保护Spring Boot应用程序安全

Posted on 2020-02-22 | | Visitors:

应用程序的安全性是要考虑的最大问题之一。

每天都有数百万用户将敏感信息发布到Internet上，而如何确保这些信息的安全是开发人员面临的最大挑战之一。幸运的是，这不是一个新问题，我们有几种工具可用来确保应用程序用户数据的安全。那些特别实用且易于实现的工具之一就是JWT（JSON Web令牌）。

描述

在整个这篇文章中，我们将学习如何使用JWT和Spring Boot实现基本的用户身份验证，但是首先我们需要知道JWT是什么。如果我们进行快速的Google搜索，我们会发现JWT是RFC 7519开放标准。如果您转到该链接，则会找到以下定义：

JSON Web令牌（JWT）是一种紧凑的，URL安全的方法，用于表示要在两方之间转移的声明。JWT中的声明被编码为JSON对象，用作JSON Web签名（JWS）结构的有效负载或JSON Web加密（JWE）结构的纯文本，从而使声明可以进行数字签名或完整性保护带有消息验证码（MAC）和/或加密的消息。*

这些定义有时看起来比它们复杂，因此我们将尝试以一种更简单的方式进行解释。在任何具有私人信息或访问受限的应用程序中，我们都需要一种方法来验证请求，而无需为每个请求处理客户端的登录凭据。为此，我们有JWT，它无需Cookie或登录信息即可验证用户身份。

说明

用户登录时，后端会生成一个JWT，其组成如下：

头部（Header）

我们可以看到JWT从头部开始，头部包含有关必须如何解释令牌的所有相关信息。

消息体（Payload）

消息体包含声明。简而言之，声明是用户的数据（或任何实体）加上为 Token 添加功能的其他重要信息（非强制性）。

我们可以找到三种类型的索赔：已注册，公开，私有。

已注册的声明\用于提供有关令牌的其他信息，例如令牌的创建时间或令牌到期时间。这些声明不是强制性的。

公开声明\由使用JWT的人定义。应该注意使用的名称，因为它们可能导致冲突。

私人声明\可以用于存储信息，而无需使用注册的声明或公共声明。请记住，它们很容易发生碰撞。
